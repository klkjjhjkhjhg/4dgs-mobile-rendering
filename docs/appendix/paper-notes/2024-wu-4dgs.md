## 方法核心(PDF §1/§2/§3/§4 直引)
1. **整体表示,非逐帧 3DGS**:不把动态场景拆成 N 个独立 3DGS;**只维护一套 canonical 3D 高斯**,对每个时间戳由 deformation field 预测形变(位姿 / 旋转 / 尺度 / 不透明度) → 内存复杂度从 `O(tN)`(逐帧)降到 `O(N + F)`(PDF §2.2 直引)。
2. **HexPlane 启发的分解式神经体素编码**:把 4D 时空体素分解为 6 个特征平面(`XY`/`XT`/`YT`/`XZ`/`YZ`/`TZ` 之类)的乘积组合,大幅降低 4D voxel grid 显存与查询复杂度(PDF §3 直引,Figure 3)。
3. **可学习变形场(Spatial-Temporal Structure Encoder + Multi-head Decoder)**:canonical 3D 高斯在每个时间戳 t 由一个轻量级 MLP 多头预测 `Δx, Δy, Δz, Δr, Δs`(位置、旋转、尺度);**本质是 canonical + deformation 范式**(PDF §4 直引,Figure 3)。
4. **邻居高斯结构化耦合(Spatial-Temporal Structure Encoder)**:变形场额外通过空间-时间结构编码把"相邻高斯"连接,使运动/形变在邻域内连贯(避免孤立形变)(PDF §4 直引)。
5. **直接用 3D Gaussian Splatting 光栅化器渲染**:每帧重建出的 3D 高斯直接走标准 splatting rasterizer,无需发明新渲染核,天然继承 3DGS 的实时性(PDF §1 + §3 直引)。
6. **自监督监督**:用光度 L1 + D-SSIM 损失 + 正则化在多视角视频序列上自监督训练,无需 GT 几何 / 运动(PDF §5.1 直引)。

## 与本调研主线的关系

### 1. 主线对标(必查)
- `docs/00-goal.md` 明确把 4DGS 原论文列为对标主线之一,排第一位。
- 本论文是 **4DGS-1K(Yuan 2025)真·对标的"上游基础"**:4DGS-1K 的 vanilla baseline `4DGS retrained` 直接对照本文 §4 的方法(PDF Table 1 of 4DGS-1K 直引)。

### 2. 借鉴价值
- **HexPlane 分解编码是后续几乎所有 4DGS / 4DGS-1K 加速路线的共同前置依赖** —— 调研时需把 4DGS 的 HexPlane 视为"原始 SOTA 训练路径",所有衍生加速都必须能落在 HexPlane 输出之上。
- **canonical + deformation 范式**意味着 4DGS 在推理时的"额外代价"集中在 deformation MLP 上,这恰好是后续做**时间复用 / mask 共享**的天然分界点(也是 4DGS-1K Temporal Filter mask 的入口)。

### 3. 不可作为移动端最终方案
- 82 FPS @ RTX 3090(桌面 24 GB)vs 目标 Adreno 8 Gen 4(手机共享显存 4~8 GB):**论文未给出移动端数字**,**反例基准**:本论文是移动端"对比上限",不是"最终方案"。

### 4. 对采集端反推(00-goal.md §"必须覆盖")
- D-NeRF synthetic(单目)+ HyperNeRF(1~2 cam)+ Neu3D(15~20 cam):训练对"高质量重建 + 多视角同步"有相当要求。**单目场景的精度天花板受限于 deformable 范式**;**Neu3D 多相机场景的精度最高**(Table 3 中 4D-GS PSNR 31.15 已经压住了同期 NeRFPlayer / HyperReel)。
