# 调研目标:4DGS 加速领域中国国内单位 Landscape(spec)

> 本文件是**调研目标 spec**,由主对话 session 写入。后续调研 subagent 在执行时,以此为目标。
> 与 `00-goal.md` 的关系:**互补不冲突**——`00-goal` 是技术深度调研(精度 + 渲染加速),本 spec 是**地理维度切片**(国内单位活跃度 + 趋势)。
> 调研产物可反哺 `00-goal`:识别出的国内单位核心工作可能给 `evolution.json` 补节点,或成为 `survey/` 的新章节。

- **平级目录**:本目录与 `01-`、`02-`、`03-`、`04-`、`05-`、`evolution/`、`survey/` 平级,但**调研维度正交**。

## 背景 / 任务定位

本研究服务于**主线 4DGS 调研项目**(高通旗舰 + Vulkan 1.3 实时渲染)的**前置情报**:
- 在跟国内单位合作 / 招聘 / 选 SDK / 选 baseline 时,需要一份**可信的国内 4DGS 活跃单位清单**
- 用于回答:"国内哪些单位在 4DGS 加速方向有持续产出?代表作是哪几篇?技术路线偏向哪边?"

**主对话中的原始诉求**(用户原话):
> "4dgs 加速领域中中国国内有哪些单位比较活跃,是否能够总结下各个单位的发展趋势和核心工作"

## 必须覆盖(按优先级)

### 第一优先级:活跃单位清单

- 至少 **15 个高校实验室 / 大厂研究院**(从公开论文 + arXiv 4DGS 主题反查作者单位)
- 每个单位至少给出:
  - **单位全称 + 所在地 + 团队负责人**(若可识别)
  - **核心工作 2-5 篇**(论文名 + 链接 + 在 4DGS 加速中的角色)
  - **技术路线归属**(对照现有 5 派系 A/B/C/D/E 之一,或说明是跨派系)
  - **趋势判断**(近 1 年研究重心是朝哪个方向迁移)

### 第二优先级:按技术路线交叉

- 表格呈现:**单位 × 技术路线**矩阵
- 帮助回答:"mesh-guided 4DGS 国内谁最强?""实时渲染加速国内谁在做?""压缩 / 量化国内谁在做?"

### 第三优先级:产业落地 / 应用场景

- 国内单位的应用场景偏好(自动驾驶 / 具身智能 / AR-VR / 数字人 / 视频会议)
- 国内大厂研究院(小米 / OPPO / 字节 / 腾讯 / 阿里 / 华为)的 4DGS 投入情况(若有)

### 第四优先级（已升级，原为"潜在合作 / 招聘候选"）：合作候选清单

> **目标**：服务主业（小米图形架构部）的**对外合作候选**调研。每个 institution 必须给出可联系的具体老师 + 可直接打开的代表论文链接。

- **轻量重叠可接受** — 不强求"少重叠"，但要标出"已经做的方向"与"还能扩展的方向"
- 每个单位至少给出：
  - **代表老师姓名 + 中文 + 英文 + 职称**（教授 / 副教授 / 助理教授 / 研究员 / 博士生导师）
  - **代表老师个人主页 / 实验室主页 URL**（可联系入口）
  - **代表作链接**（每篇论文都给 `arxiv.org/abs/{id}` 直链 + 项目主页（如有））
  - **团队规模 / 在读博士生数量**（如能从主页推断；若无法验证则标 "未知"）
  - **学生去向**（往工业界 vs 学术界；如华为 / 字节 / 大疆 / 小米等是否有校友）
- **优先级排序字段**（每个 institution 必填）：
  - `co_op_value_high`: 高度匹配（合作价值大）— 给主业的合作候选
  - `co_op_value_medium`: 中度匹配（数据 / 算法 / 应用之一有亮点）
  - `co_op_value_low`: 弱匹配（仅作 baseline 参考）
  - `co_op_value_unknown`: 信息不足，需后续手动确认
- **co_op_value_high 必选规则**：
  - 在 4DGS 表示 / 重建 / 加速 / 生成 中至少 1 项有持续产出
  - 代表老师在中国大陆 / 港澳能直接邮件联系
  - 团队规模 ≥ 3 人（独立 PI 不算，因合作难以开展）

5. **`sources.md`**(白名单 + 1-hop 引用规范)
   - 与 `05-survey-methodology.md` 保持一致

## 调研方法论(沿用 00-goal + 05)

- **白名单优先**:SIGGRAPH / CVPR / ECCV / ICCV / ICLR / NeurIPS / TPAMI / TVCG 2024-2026 的 4DGS / 3DGS 论文 → 反查第一作者 / 通讯作者单位
- **一跳引用**:不链式引用;若 paper note 提及某国内单位,需要至少一个直接验证源(论文 author affiliation 或实验室主页)
- **先谈现状**:不做未来预测,只描述已发表工作 + 公开声明的研究方向
- **A+B fact-check**:per 单位至少 2 个独立信源(论文 + 实验室主页 / 个人主页 / 公开演讲)

## 第二轮扩展：每个代表老师近 3 年所有 4DGS/3DGS 相关论文（增量调研）

> **目的**：第一轮 paper-first 抓的是 arxiv listing 关键词命中的论文，但**有些老师的工作关键词不直接出现 splatting 字样**（如综述论文、NeRF 时代延续工作、与动态场景相关的论文）。本轮反向反查通讯作者主页，把每位老师近 3 年所有 4DGS/3DGS/动态场景/神经渲染相关论文都收进来。

- **不重复收第一轮已确认的 63 篇**（去重 arxiv ID）
- **每个 institution 重点查 1-2 位代表老师**（已在第四优先级中识别的 team_lead）：
  - 查 Google Scholar 个人主页（"<name> <institution> scholar"）
  - 查 DBLP 个人页（"https://dblp.org/pid/{initials}/{pid}.html"）
  - 查 实验室主页 "publications" 段落
- **纳入标准**（任一即收）：
  - 论文标题 / 摘要 含 "Gaussian Splatting" / "3DGS" / "4DGS" / "Gaussian" + "Dynamic" / "Dynamic Scene" / "Neural Rendering"
  - 论文 venue 是 SIGGRAPH / CVPR / ECCV / ICCV / ICLR / NeurIPS / TPAMI / TVCG
  - 论文第一作者 / 通讯作者是已知 institution 的代表老师
- **排除**：明确跟 4DGS/3DGS/动态场景无关的工作（如纯分类、纯目标检测、纯图像生成）
- **目标数量级**：在现有 63 篇基础上扩到 **150-200 篇**（2-3 倍）

## 与现有 cron job 的关系

- 不新增独立 cron job,复用 `arxiv_4dgs_scan`(每周一/四/日)的输出,过滤"作者单位含 .cn / .hk / .tw / .mo"或"通讯作者为中国机构"的新论文
- `meeting_scan`(每月 1/15)扫描会议接收名单时,可同步检查国内单位的接收论文分布变化

## 不要做的事

- 不要去猜某个团队"在做但还没发"的工作
- 不要把台湾 / 新加坡 / 海外华人团队算进来(等后续扩展)
- 不要给单位评级 / 排名(只能描述事实,不做价值判断)
- 不要写论文级别的深度技术分析(那是 `00-goal` 的事,本 spec 只到"单位 + 代表作 + 趋势"粒度)
