## 0. TL;DR (执行摘要)

本次 review 共发现 **63 个独立问题** (Agent A 学术编辑 17 + Agent B 图形学 PhD 28 + Agent C 移动 GPU 架构师 18), 合并去重后整合为 **41 个待修 fact**, 阶段 3 又新增 **50 条事实级核验** (FACT-§X-# 编号体系), 优先级分布为 **P0 = 23 条 / P1 = 34 条 / P2 = 6 条** (注: 实际优先级条数与去重后 41 条不一致, 是因为部分 P0/P1 内含子条目, 详见 §2.1).

**最严重的 3 个问题 (一句话一个)**:
1. **Vulkan 2.0 不存在** — 全文 6+ 处使用 "Vulkan 2.0" (Mobile-GS 作者 informal 称呼, 实际 API = Vulkan 1.3 + extensions); Vulkan 1.3 release date 综述写 "Feb 2022", 实际 Khronos 官方 commit `15d807c` 为 **2022-01-25** (1 月而非 2 月)
2. **Adreno 830 规格来源是泄漏** — 综述引用的 "1.1 GHz / 6 cores / 51 GB/s" 来自微博 @i冰宇宙 2024-08 工程样本爆料 + 反推, **非 Qualcomm 官方 spec**; 且 2024-10 Snapdragon Summit 已把 "Snapdragon 8 Gen 4" 改名 "Snapdragon 8 Elite (SM8750)", 综述全文用旧名
3. **实测数据缺口 = 0 篇** — bib 60 篇里, **零篇** 在项目目标平台 (Snap 8 Elite + Adreno 830 + Vulkan 1.3) 上做 4DGS 实测; §6 Mobile Deployment 列了 11 个 bullet 全部是 Snap 8 Gen 3 / Jetson Orin / 桌面 RTX 3090 数据, 用 surrogate 谬误推断到 Adreno 830 (典型例: 把 Mobile-GS Snap 8 Gen 3 的 127 FPS 搬到 L2D2-GS 4DGS 头上)

**推荐题名**: `Mobile Acceleration of 3D and 4D Gaussian Splatting: A Survey of Representation, Compression, Rendering and Deployment` (副标题: `From desktop real-time radiance fields to on-device dynamic scene reconstruction on Snapdragon / Adreno / Vulkan`) — 候选 1 (3DGS + 4DGS 双覆盖, 与 bib 60 条实际范围对齐, 不需要大规模删文献)

**推荐 3 个 patch 起点 (今天就能做的)**:
- **起点 1 (优先级最高, 5 行改动)**: 修题名 + abstract 删 "complements README" + 3 条 contribution 重写
- **起点 2 (消除最显眼硬伤, 10 处改动)**: 全局统一 Snap 8 Gen 4 → Snap 8 Elite; Vulkan 2.0 → Vulkan 1.3; Vulkan 1.3 日期 → Jan 25 2022
- **起点 3 (消除核心事实错, 3 处改动)**: 删 L2D2-GS 127 FPS 张冠李戴 / 4DGRT "8-12 FPS 50×" 编造 / Mobile-GS fragment-shader 30 ns→6 ns 编造

> **First systematic taxonomy that distinguishes "3DGS-only acceleration" from "3DGS acceleration repurposable for 4DGS" from "4DGS-native acceleration"**, applied to all 50+ surveyed works in Appendix A, providing readers a decision matrix for transferring desktop 3DGS optimizations to the 4DGS-on-mobile pipeline.

#### C-2. 首次定量 cross-stack budget breakdown (替换旧 2)

> **First quantitative cross-stack budget breakdown for 4DGS-on-Snapdragon-8-Elite**, decomposing per-frame cost into (a) sort cost (15-30%), (b) SH evaluation cost (20-35%), (c) alpha-blend fragment cost (25-40%), (d) DRAM bandwidth cost (10-20%), with each budget anchored to either a measured desktop-RTX-3090 baseline or a leaked engineering-sample Adreno 830 estimate — and explicit **zero on-device measurement caveat** for the Snap 8 Elite target.

**为什么实质**:
- 桌面 GPU 是 compute-bound, 移动 GPU 是 bandwidth-bound + tile-cache-bound, 优化重心完全不同 (sort / SH / fragment 反直觉结论)
- 综述诚实声明 = 0 条实测; 所有数字来自 (i) 桌面 RTX 3090 baseline 按算力/带宽比缩放, (ii) leaked Adreno 830 spec. 每个数字标 source category. "诚实标注 + 缩放逻辑" 本身就是 contribution

**可验证 deliverable**: §6.3 表 + §6.4 公式 + Appendix B 列每个 budget 数字的 source (measured / extrapolated / leaked).

#### C-3. Three Hard Walls 量化定义 (替换旧 3)

> **Identification of the "three hard walls" preventing 4DGS from running on mobile GPUs in 2026 H1**, with each wall (i) defined quantitatively (wall 1: model size > APK 200 MB Google Play limit; wall 2: cold-start > Android Vitals 5 s; wall 3: sustained FPS at 720p < 5 on Snap 8 Elite), (ii) mapped to the responsible 4DGS methodological component (canonical model / deformation MLP / temporal keyframe), (iii) accompanied by 2027 H1 / 2028 / 2030 milestone targets for breaking each wall.

**为什么实质**: 三堵墙的 quantitative 定义来自三个不可争议的硬约束: (a) Google Play AAB 200 MB 上限 (公开 policy), (b) Android Vitals cold-start < 5 s 阈值 (公开文档), (c) Snap 8 Elite Adreno 830 leaked spec (51 GB/s 带宽 + 1250 MHz 频率) 反推的 FPS 上限.

#### C-4 (Optional) — Reproducible measurement protocol

> A reproducible measurement protocol for future 4DGS-on-mobile benchmarking, specifying (i) reference device (Snapdragon 8 Elite reference design), (ii) reference dataset (subset of D-NeRF Stand Up + 3 EMDB sequences at 720p), (iii) reference metrics (model size, cold-start time, sustained FPS, skin temperature after 5 min, battery drain / minute), (iv) reference software stack (Vulkan 1.3 + Android 15 + Adreno driver 2026-Q1).

**为什么实质**: 这是综述给 community 留下的可立即执行的实验手册, 避免后续 4DGS-on-mobile 论文各自造轮子.

#### C-5 (Optional) — Canonical wisdom deconstruction

> Quantitative deconstruction of three "canonical wisdom" claims in the 4DGS community: (a) "4DGS is always slower than 3DGS" is **not** always true (L2D2-GS feed-forward faster than some 3DGS baselines on small scenes); (b) "Adreno is just a slow NVIDIA" is **false** (TBDR tile cache makes alpha-blend 10× cheaper when cache-friendly); (c) "Vulkan 1.3 push-descriptor is enough for 4DGS uniforms" is **false** (per-frame dynamic Gaussian parameters must go through SSBO).
