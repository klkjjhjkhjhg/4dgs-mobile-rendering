\section{Datasets and Metrics}
\label{sec:datasets}

\subsection{Commonly Used Datasets}
4DGS training and evaluation rely on a handful of well-established
dynamic-scene datasets:
\begin{itemize}
  \item \textbf{D-NeRF}~\cite{pumarola2021dnerf} (synthetic): eight
        synthetic monocular scenes (e.g.\ Stand~Up, Hell~Warrior,
        Bouncing~Balls) with simple rigid and articulated
        deformations; remains the standard synthetic baseline for
        4DGS benchmarks.
  \item \textbf{DyCheck}~\cite{gao2022dycheck} (iPhone): real-world
        dynamic scenes captured with a handheld iPhone, with
        held-out multi-view ground truth; the de-facto real-world
        benchmark for 4DGS quality evaluation.
  \item \textbf{Plenoptic Video}~\cite{li2022plenvdb} (Google): a
        12-camera array dataset (\emph{Ballroom}, \emph{Cut~Roasted
        Beef}, etc.) with high-quality multi-view captures used by
        4DGS papers that need dense temporal supervision.
  \item \textbf{Nerfies}~\cite{park2021nerfies} and
        \textbf{HyperNeRF}~\cite{hypernerf2022}: monocular
        dynamic humans and deformable objects captured with a
        handheld camera; the canonical datasets for
        monocular-dynamic 4DGS evaluation.
  \item \textbf{CMU Panoptic}~\cite{simon2017cmu}: multi-view,
        multi-person interactions captured by a 31-camera dome;
        the standard multi-person benchmark for 4DGS in crowd
        and pose scenarios.
\end{itemize}

\subsection{Evaluation Metrics}
\begin{itemize}
  \item \textbf{Quality:} PSNR / SSIM / LPIPS (per frame).
  \item \textbf{Temporal consistency:} Temporal LPIPS / flow warping
        error.
  \item \textbf{Speed:} FPS (decode + render) / training time
        (minutes).
  \item \textbf{Storage:} model size (MB) / peak VRAM (MB).
  \item \textbf{Mobile-specific:} power (mW) / thermal stability
        (sustained FPS).
\end{itemize}
