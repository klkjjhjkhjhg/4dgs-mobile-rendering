## 0. 一句话总结

过去 12 个月（25 H2 ~ 26 H1）4DGS 加速论文出现**显著分化**，从单点 prune / quant 扩展到端到端 pipeline 改造。本项目（Snap 8 Gen 4 + Vulkan 1.3 + ~300 万 splat 限制）**不直接对标任何现有论文的硬件**，但能从 4 个分化方向各取一段。

## 1. 现有 4 条工程路径

### 路径 A：表示分离（canonical + deformation 系）

**代表论文**：Wu 4DGS (2310.08528) / Yang Deformable 3DGS (2309.13101) / 4D-RotorGS (2402.03306)

- **核心思想**：把"跨帧不变的几何"装进 canonical set，"每帧变化"装进 deformation field
- **工程难点**：canonical 优化时容易塌到 trivial solution（不分离）；deformation field 在 mobile 上要每帧 forward 一次推理
- **mobile 直接对标难度**：⭐⭐（中等）—— Wu 4DGS 在 RTX 3090 上 5-10 FPS，离移动端 30 FPS 目标差 6 倍
- **可借鉴**：canonical set 可离线算好 + cache，每帧只算 deformation

### 路径 B：时序复用（4DGS-1K 系）

**代表论文**：4DGS-1K (2503.16422) / SpeeDe3DGS (2506.07917) / SharpTimeGS (2602.02989) / OMG4 (2510.03857) / StreamSTGS (2511.06046) / 4DGS-CC (2504.18925) / PD-4DGS (2605.11427)

- **核心思想**：给每个 splat 打"这一帧还活跃不活跃"的分数，跨帧复用 inactive 子集
- **工程难点**：score 计算开销（额外一个 forward pass）；跨 30 帧的 hash 表冲突率
- **mobile 直接对标难度**：⭐（最简单）—— 4DGS-1K TITAN X 上 200+ FPS（2015 年显卡）
- **可借鉴**：STV 评分思路 + cross-frame visibility cache

### 路径 C：动静态显式分离（25-26 主线）

**代表论文**：SharpTimeGS (2602.02989) / OMG4 (2510.03857) / CAGS (2605.09279) / SpeeDe3DGS (2506.07917) / 4DGS-CC (2504.18925) / PD-4DGS (2605.11427)

- **核心思想**：直接把 splat 集切成"static set" + "dynamic set"，**跨帧只重渲 dynamic set**
- **工程难点**：splat 该归哪一边的判定；dynamic set 的 refit 策略
- **mobile 直接对标难度**：⭐⭐（中等）—— driving 系论文对"动静态分离"的语义表达最直接
- **可借鉴**：最直接对应"动静态分离"直觉；详见 [docs/01-high-precision-representation.md §6](01-high-precision-representation.md#6-静态-动态分离)

### 路径 D：流式 streaming（mobile 落地系）

**代表论文**：AirGS (2512.20943) / StreamSTGS (2511.06046) / 4DGCPro (2509.17513) / PD-4DGS (2605.11427)

- **核心思想**：把 4DGS scene streaming 到设备端，增量更新而非全场景重传
- **工程难点**：mobile 网络带宽；on-device 重建；storage 不是问题、latency 是
- **mobile 直接对标难度**：⭐（最直接）—— 系 mobile streaming 报告
- **可借鉴**：mobile 上的端云协同 + 增量更新思路，对 100ms 内渲染目标友好

## 2. 25 H2 ~ 26 H1 的 5 个新趋势

| 趋势 | 标志论文 | 含义 |
|---|---|---|
| **时序压缩成主线** | 4DGS-CC / StreamSTGS / SpeeDe3DGS / SharpTimeGS / OMG4 / PD-4DGS（6 套独立方法同做时序压缩） | 25 年起 "帧间冗余压缩" 成独立研究路线 |
| **量化 + 帧间组合** | 4DGS-1K（VQ + 帧间 mask） / GETA-3DGS（structured prune + quant） | 24 年是 "quantize alone" / "prune alone"，25 年组合方案胜出 |

| **canonical rotation 思路复活** | 4D-RotorGS（2402.03306）等 | 4D-RotorGS 的 canonical rotation 是动静态分离的工程化身，25 年起被多个新工作采纳 |
| **Survey 集中在 25-26** | SUCCESS-GS（2512.07197, 37 页） | = **领域成熟度信号**（该写综述 = 该沉淀下来了） |

## 3. 与"动静态分离"主线的交叉视角

> 这点容易被忽略，特地单列。

**5 个新趋势中有 3 个本质上是"动静态分离"思想**：

- 趋势 1（时序压缩） = 路径 B = 不分离表示，但**跨帧复用**
- 趋势 2（量化 + 帧间） = 路径 B + path C 组合 = **static 部分量化 frozen, dynamic 部分量化 adaptive**
- 趋势 4（canonical rotation 复活） = 路径 C 工程化

→ **动静态分离不是单条路线，是 25 H2 ~ 26 H1 加速研究的隐身主角**。  
→ 把它单列主线，是 [§01 §6](01-high-precision-representation.md#6-静态-动态分离) 的关键 insight。

## 5. 给读者的术语速查（30+ 条）

> 见 [README.md §3 术语词典](../README.md)。

**5 个词必懂**：
- **Splat** = 一个"喷溅点"（高斯椭球）
- **STV** = Spatial-Temporal Variation Score（4DGS-1K 的复用评分）
- **canonical set** = 跨帧不变的几何
- **deformation field** = 把 canonical 变到当前帧的形变场
- **Vulkan 1.3** = 高通旗舰支持的图形 API 版本（Mobile-GS 用 Vulkan 2.0）
